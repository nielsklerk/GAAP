\chapter{Results}
This chapter begins with simulation results that demonstrate the reliability and limitations of the \gls{gaap} algorithm for determining the flux values of different sources. Following this, the flux measurements of the \gls{mer} tiles are discussed and compared to the fluxes in the \gls{mer} catalog.

\section{Simulation}

\subsection{Deconvolution}\label{subsec: Deconvolution}
The deconvolution is stable when the weight is 'larger' than the \gls{psf}. In the simulation, this is straightforward to derive because both are Gaussian, so comparing their scale factors yields a direct measure of their relative sizes. The convolution of two Gaussians gives a Gaussian with a variance that is the sum of the variances of the original Gaussians. So the deconvolution of a weight function with a variance of $\sigma_\mathrm{weight}^2$ with a \gls{psf} with a variance of $\sigma_\mathrm{PSF}^2$ gives the deconvolved weight with a variance $\sigma^2_\mathrm{deconv}=\sigma^2_\mathrm{weight}-\sigma^2_\mathrm{PSF}$. From this, it is clear that the deconvolution fails when the weight is smaller than the \gls{psf}. 

Furthermore, due to the limited image size, larger weights relative to the image size begin to introduce artifacts in the deconvolution. To prevent these artifacts, the size of the image can be temporarily increased for the deconvolution and then again truncated to the correct size.

The figure shows 2 regions where the measured flux values are inaccurate. The first region is the measurements taken with weights smaller than the \gls{psf}. This result is expected, as shown in \autoref{subsec: Deconvolution}. The second region comprises measurements above the \gls{psf} line and is $\gtrsim10$ pixels wide. These incorrect measurements are due to the limited size of the weight function, leading to fringing. This can be mitigated by increasing the image size of the weight function.

\subsection{Aperture Shape}
The \gls{snr} for different weight function shapes was calculated. The results are shown in \autoref{fig: best aperture}.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\linewidth]{results/figures/simulation/aperture_size.pdf}
    \caption{The SNR for different combinations of the horizontal and vertical standard deviations of the weight. The dashed line shows the spherical Gaussians; all the off-diagonal combinations are elliptical weights.}
    \label{fig: best aperture}
\end{figure}

The best weight is approximately equal to the intrinsic shape of the galaxy. This holds for both circular and elliptical shapes. Thus, the weight that yields the highest \gls{snr} is the one that best fits the intrinsic image. This can be estimated by fitting to an observation in a single field, preferably at the highest resolution, and then identifying the corresponding pre-seeing weight to apply to all bands.

\section{Observation}
 

\subsection{Preprocessing}
The \glspl{psf} of tile \textbf{TILEINDEX} is used to find a relation between the size of the source in the VIS band and the size of the weight function that optimizes the \gls{snr} of the flux in all the bands. A Gaussian object is simulated with Gaussian noise. Different sizes of weight functions are used to find the \gls{snr} for each of the \glspl{psf}. For each of the \glspl{psf}, the curve is divided by the maximum, and then they are added together to get a single curve capturing the combined \gls{snr} for all the \glspl{psf}. The size of the weight function where this curve is maximized is combined with the size of the source. This is repeated for different sizes of the source, resulting in a relation between the size of the source and the size of the weight function that maximizes the \gls{snr}. A 5th-order polynomial is fitted to the measurements. The result is shown in \autoref{fig: relation pre post seeing}. 

\begin{figure}[H]
    \centering
    \includegraphics[width=0.5\linewidth]{results/figures/observation/relation_pre_seeing_post_seeing.pdf}
    \caption{The relation between the size of the Gaussian weight function that maximizes some form of the \gls{snr} over all bands compared to the size of the source in the VIS field. }
    \label{fig: relation pre post seeing}
\end{figure}
Next, a Gaussian is fitted to the sources in the \gls{mer} tiles, which is used to find the best weight size to optimize the \gls{snr}. The coordinates in the \gls{mer} catalog are used to set the centers of the object, which will also be used for the flux measurements. The sizes of the weight function are clipped between 0.7" and 1.2". The lower bound is to make sure that the weight function is bigger than the largest \gls{psf}. The upper bound is to prevent the fringing effect of the deconvolution.
